## üîß Prerequisites Setup

### Azure Subscription Requirements

#### Step 1: Azure Account Setup
1. **Create Azure Account**: Visit [Azure Portal](https://portal.azure.com)
2. **Subscription**: Active Azure subscription (Free tier available)
3. **Billing**: Valid payment method (required even for free tier)
4. **Region**: Choose region closest to your location

#### Step 2: Required Azure Services

##### Azure OpenAI Service Setup
```bash
# Check available regions for Azure OpenAI
az cognitiveservices account list-kinds --query "[?contains(kind, 'OpenAI')]"
```

1. **Navigate** to Azure OpenAI in the Azure Portal
2. **Create** a new Azure OpenAI resource
3. **Select** appropriate pricing tier:
   - **Standard S0**: Pay-per-use (recommended for development)
   - **Provisioned**: Fixed capacity (for production)
4. **Deploy Models**:
   - DALL-E 3 (dall-e-3)
   - GPT-4 Vision (gpt-4-vision-preview)

##### Azure AI Foundry Access
1. **Visit** [Azure AI Foundry](https://ai.azure.com)
2. **Create Project** or use existing one
3. **Enable FLUX Models**:
   - FLUX.1 [dev]
   - FLUX.1 [schnell]
4. **Configure Endpoints** for model access

#### Step 3: API Key Generation

##### For Azure OpenAI:
```bash
# Using Azure CLI
az cognitiveservices account keys list \
  --name "your-openai-resource" \
  --resource-group "your-resource-group"
```

##### Manual Steps:
1. **Navigate** to your Azure OpenAI resource
2. **Go to** "Keys and Endpoint" section
3. **Copy** Key 1 or Key 2
4. **Note** the endpoint URL
5. **Save** securely (never commit to version control)

#### Step 4: Permissions & Roles

Required Azure roles:
- **Cognitive Services User**: For Azure OpenAI access
- **AI Developer**: For Azure AI Foundry
- **Contributor**: For resource management (optional)

```bash
# Assign roles using Azure CLI
az role assignment create \
  --assignee "your-username" \
  --role "Cognitive Services User" \
  --scope "/subscriptions/your-subscription-id"
```

### Docker Setup

#### Step 1: Create Dockerfile

Create `Dockerfile` in project root:
```dockerfile
# Multi-stage build for optimal size
FROM node:20-alpine AS deps
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app

ENV NODE_ENV production

RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

COPY --from=builder /app/public ./public
COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs

EXPOSE 3000
ENV PORT 3000

CMD ["node", "server.js"]
```

#### Step 2: Create Docker Compose

Create `docker-compose.yml`:
```yaml
version: '3.8'

services:
  azure-image-studio:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - AZURE_API_KEY=${AZURE_API_KEY}
      - AZURE_API_BASE_URL=${AZURE_API_BASE_URL}
    env_file:
      - .env.production
    restart: unless-stopped
    volumes:
      - ./src/app/config:/app/src/app/config:ro
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000/api/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s

volumes:
  app_data:
```

#### Step 3: Build and Run

```bash
# Build Docker image
docker-compose build

# Start services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop services
docker-compose down
```

## üõ†Ô∏è Troubleshooting

### Common Installation Issues

#### Node.js Version Issues

**Problem**: `npm ERR! peer dep missing: react@^18.0.0`

**Solution**:
```bash
# Check current Node.js version
node --version

# If version is too old, update
# Windows: Download from nodejs.org
# macOS: brew upgrade node
# Linux: Use package manager or nvm

# Clear and reinstall dependencies
rm -rf node_modules package-lock.json
npm install
```

#### npm Permission Issues (macOS/Linux)

**Problem**: `EACCES: permission denied`

**Solution**:
```bash
# Fix npm permissions
sudo chown -R $(whoami) ~/.npm

# Or use nvm (recommended)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install --lts
nvm use --lts
```

#### Port Already in Use

**Problem**: `Error: listen EADDRINUSE :::3000`

**Solution**:
```bash
# Find process using port 3000
lsof -i :3000
# or on Windows
netstat -ano | findstr :3000

# Kill the process (replace PID)
kill -9 <PID>
# or on Windows
taskkill /PID <PID> /F

# Or use different port
npm run dev -- -p 3001
```

#### Build Failures

**Problem**: Build fails with TypeScript errors

**Solution**:
```bash
# Clear Next.js cache
rm -rf .next

# Clear node_modules
rm -rf node_modules package-lock.json
npm install

# Check TypeScript configuration
npm run typecheck

# Build again
npm run build
```

### Azure Configuration Issues

#### Invalid API Key

**Problem**: `401 Unauthorized` or `Invalid API Key`

**Checklist**:
- [ ] API key is correct and complete
- [ ] Key has proper permissions
- [ ] Endpoint URL matches resource
- [ ] API version is supported
- [ ] Resource is in correct region

**Solution**:
```bash
# Test API key manually
curl -H "Authorization: Bearer YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     "https://your-resource.openai.azure.com/openai/deployments?api-version=2024-10-01-preview"
```

#### Endpoint Configuration Issues

**Problem**: `404 Not Found` or connection errors

**Checklist**:
- [ ] Endpoint URL is correctly formatted
- [ ] Deployment names match Azure configuration
- [ ] API version is supported by deployment
- [ ] Network connectivity to Azure

**Solution**:
```bash
# Verify endpoint accessibility
curl -I "https://your-resource.openai.azure.com"

# Check deployment exists
az cognitiveservices account deployment list \
  --name "your-resource" \
  --resource-group "your-rg"
```

#### Model Not Available

**Problem**: Model not showing in UI or API errors

**Checklist**:
- [ ] Model is deployed in Azure
- [ ] Model has sufficient quota
- [ ] Deployment name matches configuration
- [ ] Model is enabled in configuration files

### Runtime Issues

#### Memory Issues

**Problem**: Application crashes with out-of-memory errors

**Solution**:
```bash
# Increase Node.js memory limit
export NODE_OPTIONS="--max-old-space-size=4096"
npm run dev

# Or add to package.json scripts:
"dev": "NODE_OPTIONS='--max-old-space-size=4096' next dev --turbopack"
```

```bash
# Windows: Task Manager

# Check Node.js performance
```
